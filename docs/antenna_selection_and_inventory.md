# アンテナ選択・順次Inventory確認メモ

## 目的

このメモは、UTR-SUN02-4CHおよびUTR-SUN02-8CHで、アンテナ接続状態を確認し、Inventory対象アンテナを選択して読み取りを行うための考え方をまとめたものです。

主な目的は以下です。

- アンテナ接続状態を確認する。
- 接続OKアンテナだけをInventory候補にする。
- コマンドモード用パラメータだけを一時変更する。
- FLASHを書き換えない。
- 複数アンテナ選択時は、同時指定ではなく順次切替でInventoryする。
- ANT別の読み取り結果を比較しやすくする。

## 対象機種

### UTR-SUN02-4CH / USM02

```text
UTR-SUN02-4CH
- ANT0: 内蔵アンテナ
- ANT1: 外付けアンテナ1
- ANT2: 外付けアンテナ2
- ANT3: 外付けアンテナ3
```

標準入口 `src/utr_usb_sample.py` の中で、必要に応じてアンテナ接続確認と順次Inventoryを行います。

```powershell
py .\src\utr_usb_sample.py
```

### UTR-SUN02-8CH / USM08

```text
UTR-SUN02-8CH
- ANT1〜ANT8: 外付けアンテナ
```

8CH機は4CH機とアンテナ番号体系が異なるため、専用CLIを使います。

```powershell
py .\src\utr_8ch_sequential_inventory_cli.py
```

## 使用する主なコマンド

| 処理 | コマンド | 用途 |
|---|---|---|
| アンテナ接続確認 | `55 44` | UHF_CheckAntenna。指定アンテナの接続状態を確認する |
| 4CH アンテナ切替設定読み取り | `55 43 00` | コマンドモード用パラメータやFLASHデータを読み取る |
| 4CH アンテナ切替設定書き込み | `55 33 00` | コマンドモード用パラメータを一時変更する |
| 8CH 使用アンテナ番号設定 | `55 38 00` | UTR-SUN02-8CHの使用アンテナ番号をコマンドモード用パラメータへ一時設定する |

## UHF_CheckAntenna

UHF_CheckAntennaは、アンテナ接続状態を確認するためのコマンドです。

送信データ部は以下です。

```text
44h + antenna_number
```

応答ステータスは以下です。

| ステータス | 内容 |
|---|---|
| `00h` | 接続OK |
| `01h` | 接続エラー |
| `02h` | パラメータエラー |

## 4CH機の番号体系

4CH機では、以下の番号を使います。

| 番号 | アンテナ |
|---|---|
| `00h` | ANT0 内蔵アンテナ |
| `01h` | ANT1 外付けアンテナ1 |
| `02h` | ANT2 外付けアンテナ2 |
| `03h` | ANT3 外付けアンテナ3 |

4CH機では、Inventory候補にするアンテナは次の2条件を満たすものだけです。

1. UHF_CheckAntennaで接続OKであること。
2. FLASHデータで使用許可されていること。

例えば、ANT0とANT1が接続OK、FLASHでAnt0〜Ant3が許可されている場合、候補は以下になります。

```text
Inventoryに使用できるアンテナ候補:
[0] ANT0（内蔵アンテナ）
[1] ANT1（外付けアンテナ1）
```

入力方法です。

| 入力 | 動作 |
|---|---|
| `0` | ANT0だけでInventoryする |
| `1` | ANT1だけでInventoryする |
| `0,1` | ANT0でInventory後、ANT1へ切り替えてInventoryする |
| `all` | 候補に表示されたアンテナをすべて順番に使う |
| `q` | アンテナ設定を変更しない |

4CH機では、Inventory終了時に、プログラム開始時に読み取ったコマンドモード用アンテナ設定へ戻します。

```text
コマンドモード用アンテナ設定を元に戻します。
復元対象: Ant0
FLASHは変更していません。
コマンドモード用アンテナ設定を元に戻しました。
```

## 8CH機の番号体系

UTR-SUN02-8CHでは、UHF_CheckAntennaで使う番号と、使用アンテナ番号設定で使う番号の体系が異なります。

### UHF_CheckAntenna番号

| ANT | UHF_CheckAntenna番号 |
|---|---|
| ANT1 | `00h` |
| ANT2 | `01h` |
| ANT3 | `02h` |
| ANT4 | `03h` |
| ANT5 | `04h` |
| ANT6 | `05h` |
| ANT7 | `06h` |
| ANT8 | `07h` |

### 使用アンテナ番号

| ANT | 使用アンテナ番号 |
|---|---|
| ANT1 | `00h` |
| ANT2 | `20h` |
| ANT3 | `40h` |
| ANT4 | `60h` |
| ANT5 | `80h` |
| ANT6 | `A0h` |
| ANT7 | `C0h` |
| ANT8 | `E0h` |

実機確認済みのフレーム例です。

```text
ANT1: 02 00 55 04 38 00 00 00 03 96 0D
ANT3: 02 00 55 04 38 00 02 00 03 98 0D
```

## 8CH順次Inventoryの入力方法

UTR-SUN02-8CH専用CLIでは、接続OKアンテナだけが候補に表示されます。

```text
[1] ANT1（使用アンテナ番号: 00h）
[3] ANT3（使用アンテナ番号: 40h）
選択例: 1 / 1,3 / all / q
```

入力方法です。

| 入力 | 動作 |
|---|---|
| `1` | ANT1だけでInventoryする |
| `3` | ANT3だけでInventoryする |
| `1,3` | ANT1でInventory後、ANT3へ切り替えてInventoryする |
| `all` | 候補に表示されたアンテナをすべて順番に使う |
| `q` | 対象選択を中止する |

## 順次切替方式

複数アンテナを選択した場合、同時に複数アンテナを有効化するのではなく、1本ずつ切り替えます。

例: 4CHで `0,1`

```text
ANT0へ一時変更
↓
Inventory
↓
ANT1へ一時変更
↓
Inventory
```

例: 8CHで `1,3`

```text
ANT1 使用アンテナ番号設定
↓
Inventory
↓
ANT3 使用アンテナ番号設定
↓
Inventory
```

## なぜ同時指定ではなく順次切替にするのか

同時に複数アンテナを指定すると、機種や設定条件によってNACKや想定外動作になる可能性があります。

順次切替方式にすることで、以下の利点があります。

- 現在どのアンテナで読んでいるか分かりやすい。
- ANT別の読み取り差を比較しやすい。
- FLASHを書き換えずに確認できる。
- 設定変更の影響範囲をコマンドモード用パラメータに限定できる。
- 4CH機と8CH機のアンテナ番号体系の混同を避けやすい。

## 4CH実機確認済みの流れ

### 接続確認

```text
ANT0（内蔵アンテナ）: 接続OK
ANT1（外付けアンテナ1）: 接続OK
ANT2（外付けアンテナ2）: 接続エラー
ANT3（外付けアンテナ3）: 接続エラー
```

### 入力

```text
Inventoryに使用するアンテナ番号を入力してください（例: 0,1 / all、終了は'q'）: all
Inventory対象アンテナ: ANT0, ANT1
```

### 終了時の復元

```text
コマンドモード用アンテナ設定を元に戻します。
復元対象: Ant0
FLASHは変更していません。
コマンドモード用アンテナ設定を元に戻しました。
```

## 8CH実機確認済みの流れ

### 接続確認

```text
アンテナ接続チェック結果:
  接続OK: ANT1, ANT3
補足: この時点ではアンテナ設定を書き換えていません。
```

### 入力

```text
8CH Inventoryに使用するANT番号を入力してください（終了は'q'）: all
選択順: ANT1 -> ANT3
```

### ANT1でInventory

```text
--- 使用アンテナ番号設定: ANT1 ---
送信フレーム: 02 00 55 04 38 00 00 00 03 96 0D
使用アンテナ番号設定コマンド: ACK
設定対象: ANT1 / 使用アンテナ番号 00h
```

### ANT3でInventory

```text
--- 使用アンテナ番号設定: ANT3 ---
送信フレーム: 02 00 55 04 38 00 02 00 03 98 0D
使用アンテナ番号設定コマンド: ACK
設定対象: ANT3 / 使用アンテナ番号 40h
```

### 結果まとめ

```text
Inventory実行回数: 4
合計読み取り時間: 0.56 秒
合計読み取り枚数: 29 枚
ANT別結果:
  ANT1: 読み取り 2 枚 / Inventory実行 2 回 / スキップ 0 回
  ANT3: 読み取り 27 枚 / Inventory実行 2 回 / スキップ 0 回
```

### 終了時の使用アンテナ番号戻し

```text
最後に使用したANT: ANT3
終了前に使用アンテナ番号を戻しますか？ [y/N]: y
終了前に戻すANT番号を入力してください（中止は'q'）: 1
送信フレーム: 02 00 55 04 38 00 00 00 03 96 0D
戻し完了: ANT1 / 使用アンテナ番号 00h
```

## 注意点

- FLASHは変更しません。
- コマンドモード用パラメータだけを一時変更します。
- 強制終了した場合、復元処理または戻し処理まで到達しない可能性があります。
- 確認作業では、原則として`q`で終了してください。
- 実測PC/UII、タグID、シリアル番号、顧客情報、未マスクログは公開用ドキュメントに含めないでください。
- 4CH機と8CH機はアンテナ番号体系が異なるため、処理を混同しないでください。
- UTR-SUN02V-8CHは内部アンテナ番号体系が異なるため、UTR-SUN02-8CH用CLIの実送信対象にはしません。
