# アンテナ選択・順次Inventory確認メモ

## 目的

このメモは、UTR-SUN02-4CHでアンテナ接続状態を確認し、Inventory対象アンテナを選択して読み取りを行うための手順をまとめたものです。

主な目的は以下です。

- ANT0〜ANT3の接続状態を確認する。
- 接続OKかつFLASHで使用許可されているアンテナだけをInventory候補にする。
- コマンドモード用パラメータだけを一時変更する。
- FLASHを書き換えずに、終了時に元のコマンドモード設定へ復元する。
- 複数アンテナ選択時は、同時指定ではなく順次切替でInventoryする。

## 対象機種

現時点の対象は、UTR-SUN02-4CHです。

```text
UTR-SUN02-4CH
- ANT0: 内蔵アンテナ
- ANT1: 外付けアンテナ1
- ANT2: 外付けアンテナ2
- ANT3: 外付けアンテナ3
```

8CH機はアンテナ番号体系が異なるため、本処理の対象外です。

## 使用する主なコマンド

| 処理 | コマンド | 用途 |
|---|---|---|
| アンテナ接続確認 | `55 44` | UHF_CheckAntenna。指定アンテナの接続状態を確認する |
| アンテナ切替設定読み取り | `55 43 00` | コマンドモード用パラメータやFLASHデータを読み取る |
| アンテナ切替設定書き込み | `55 33 00` | コマンドモード用パラメータを一時変更する |

## UHF_CheckAntenna

UHF_CheckAntennaは、アンテナ接続状態を確認するためのコマンドです。

送信データ部は以下です。

```text
44h + antenna_number
```

4CH機では、以下の番号を使います。

| 番号 | アンテナ |
|---|---|
| `00h` | ANT0 内蔵アンテナ |
| `01h` | ANT1 外付けアンテナ1 |
| `02h` | ANT2 外付けアンテナ2 |
| `03h` | ANT3 外付けアンテナ3 |

応答ステータスは以下です。

| ステータス | 内容 |
|---|---|
| `00h` | 接続OK |
| `01h` | 接続エラー |
| `02h` | パラメータエラー |

## アンテナ切替設定の読み取り

Pythonサンプルでは、Inventory前に必要に応じてアンテナ切替設定を読み取ります。

読み取り対象は以下です。

| パラメータ種別 | 内容 |
|---|---|
| `00h` | コマンドモード用パラメータ |
| `02h` | FLASHデータ |

実機確認例です。

```text
[コマンドモード用パラメータ]
アンテナ切替方式: 制御する
アンテナID出力: 有効
使用するアンテナ: Ant0

[FLASHデータ]
アンテナ切替方式: 制御する
アンテナID出力: 有効
使用するアンテナ: Ant0, Ant1, Ant2, Ant3
```

## Inventory候補の条件

Inventory候補にするアンテナは、次の2条件を満たすものだけです。

1. UHF_CheckAntennaで接続OKであること。
2. FLASHデータで使用許可されていること。

例えば、ANT0とANT1が接続OK、FLASHでAnt0〜Ant3が許可されている場合、候補は以下になります。

```text
Inventoryに使用できるアンテナ候補:
[0] ANT0（内蔵アンテナ）
[1] ANT1（外付けアンテナ1）
```

## 入力方法

複数アンテナ選択に対応しています。

| 入力 | 動作 |
|---|---|
| `0` | ANT0だけでInventoryする |
| `1` | ANT1だけでInventoryする |
| `0,1` | ANT0でInventory後、ANT1へ切り替えてInventoryする |
| `all` | 候補に表示されたアンテナをすべて順番に使う |
| `q` | アンテナ設定を変更しない |

## 順次切替方式

複数アンテナを選択した場合、同時に複数アンテナを有効化するのではなく、1本ずつ切り替えます。

例: `0,1`

```text
ANT0へ一時変更
↓
Inventory
↓
ANT1へ一時変更
↓
Inventory
```

例: `all` でANT0とANT1が候補の場合

```text
ANT0
↓
ANT1
↓
ANT0
↓
ANT1
```

## なぜ同時指定ではなく順次切替にするのか

同時に複数アンテナを指定すると、機種や設定条件によってNACKや想定外動作になる可能性があります。

順次切替方式にすることで、以下の利点があります。

- 現在どのアンテナで読んでいるか分かりやすい。
- ANT別の読み取り差を比較しやすい。
- FLASHを書き換えずに確認できる。
- 設定変更の影響範囲をコマンドモード用パラメータに限定できる。
- 8CH機とのアンテナ番号体系の混同を避けやすい。

## 実機確認済みの流れ

### 接続確認

```text
ANT0（内蔵アンテナ）: 接続OK
ANT1（外付けアンテナ1）: 接続OK
ANT2（外付けアンテナ2）: 接続エラー
ANT3（外付けアンテナ3）: 接続エラー
```

### 入力

```text
Inventoryに使用するアンテナ番号を入力してください（例: 0,1 / all、終了は'q'）: all
Inventory対象アンテナ: ANT0, ANT1
```

### ANT0でInventory

```text
--- Inventory対象: ANT0（内蔵アンテナ） ---
コマンドモード用アンテナ設定はすでに ANT0 です。そのままInventoryに使用します。
```

### ANT1へ切替

```text
--- Inventory対象: ANT1（外付けアンテナ1） ---

コマンドモード用アンテナ設定を一時変更します。
変更前: Ant0
変更後: ANT1（外付けアンテナ1）
FLASHは変更しません。
コマンドモード用アンテナ設定を一時変更しました。
```

### ANT1からANT0へ戻す

```text
--- Inventory対象: ANT0（内蔵アンテナ） ---

コマンドモード用アンテナ設定を一時変更します。
変更前: Ant1
変更後: ANT0（内蔵アンテナ）
FLASHは変更しません。
コマンドモード用アンテナ設定を一時変更しました。
```

## 終了時の復元

Inventoryを`q`で終了すると、プログラム開始時に読み取ったコマンドモード用アンテナ設定へ戻します。

```text
コマンドモード用アンテナ設定を元に戻します。
復元対象: Ant0
FLASHは変更していません。
コマンドモード用アンテナ設定を元に戻しました。
```

## 注意点

- FLASHは変更しません。
- コマンドモード用パラメータだけを一時変更します。
- 強制終了した場合、復元処理まで到達しない可能性があります。
- 確認作業では、原則として`q`で終了してください。
- 実測PC/UII、タグID、シリアル番号、顧客情報、未マスクログは公開用ドキュメントに含めないでください。
- 8CH機のアンテナ番号体系は4CH機と異なるため、別途実装・確認してください。
