# Field Support Quick Start

## このドキュメントの対象者

このドキュメントは、次の人が UTR-S201/202 系リーダライタを USB 接続で確認するときの入口です。

- フィールドサポート担当者
- Python 初心者
- UTR-S201/202 を USB 接続で確認したい人
- UTR-SUN02-4CH のアンテナ接続確認やANT別読み取り確認をしたい人

## 事前準備

作業前に、以下を準備してください。

- Windows PC
- UTR-S201/202 系リーダライタ、または UTR-SUN02-4CH
- USB 接続
- アンテナ
- RF タグ
- UTRRWManager での事前確認

UTRRWManager で接続、ROM バージョン確認、タグ読み取りなどの基本動作を先に確認してから、Python サンプルでの確認に進んでください。

## 最初に確認すること

- COM ポートが Windows 上で認識されていること
- Python サンプルの COM ポート選択では、表示された番号または `COM6` のような COM 名を入力できること
- 通信速度入力で未入力なら 115200 bps になること
- 通信速度が機器設定と一致していること
- 機器電源が入っていること
- アンテナが正しく接続されていること
- 送信出力設定が想定どおりであること
- 送信出力値は `dBm * 10` を LSB/MSB の順で解析すること
- 読み取りに使うタグが準備されていること

## 実行前の注意

- `UHF_SET_INVENTORY_PARAM` は自動送信しないでください。
- 設定変更系コマンドは不用意に実行しないでください。
- 実機通信は、接続条件と手順を確認した後に行ってください。
- 顧客情報、実シリアル番号、実タグ ID、未マスクの実測ログは記録や共有時に混入させないでください。
- Inventory結果は `inventory_results.txt`、`inventory_results.csv`、`inventory_results.json` に保存されます。
- CSVはExcelでの確認向け、JSONは後続処理や比較用です。
- 実測PC/UIIやログを公開する場合は、必ずマスクしてください。
- 読み取り前にブザー通知のON/OFFを選べます。ONの場合、タグありはピッピッピ、タグなしはピーで通知します。
- `q` で読み取りを終了しても、終了ブザーは鳴りません。

## よく使う確認

| 確認 | 用途 |
|---|---|
| `ROM_VERSION_CHECK` | リーダライタと通信できるか、機器応答が返るかを確認する |
| `COMMAND_MODE_SET` | コマンドモードへ移行できるかを確認する |
| `UHF_GET_INVENTORY_PARAM` | Inventory 関連パラメータを読み出して確認する |
| `UHF_INVENTORY` | タグ読み取り応答を確認する |
| `UHF_BUZZER_pipipi` | ACK 応答と実音で簡易動作を確認する |
| `UHF_CheckAntenna` | UTR-SUN02-4CH のANT0〜ANT3接続状態を確認する |
| アンテナ切替設定読み取り | コマンドモード用パラメータとFLASHデータのアンテナ設定を確認する |
| アンテナ切替設定書き込み | コマンドモード用パラメータだけを一時変更する。FLASHは変更しない |

## UTR-SUN02-4CHのアンテナ確認と順次Inventory

UTR-SUN02-4CHでは、Pythonサンプル実行中にアンテナ設定表示・接続チェックを実行できます。

この機能は、以下を目的にしています。

- ANT0〜ANT3の接続状態を確認する。
- Inventoryに使えるアンテナだけを候補表示する。
- コマンドモード用アンテナ設定だけを一時変更する。
- FLASHを書き換えずに、終了時に元のコマンドモード設定へ戻す。
- 複数アンテナを同時指定ではなく、順次切替でInventoryする。

### 実行時の入力例

```text
アンテナ設定表示・接続チェックを実行しますか？ [y/N]: y
```

ANT0とANT1が接続OKの場合、次のように候補が表示されます。

```text
Inventoryに使用できるアンテナ候補:
[0] ANT0（内蔵アンテナ）
[1] ANT1（外付けアンテナ1）
複数選択できます。例: 0 / 1 / 0,1 / all
複数選択時は、同時使用ではなく順番にアンテナを切り替えてInventoryします。
```

入力例は以下です。

| 入力 | 動作 |
|---|---|
| `0` | ANT0だけでInventoryする |
| `1` | ANT1だけでInventoryする |
| `0,1` | ANT0でInventory後、ANT1へ切り替えてInventoryする |
| `all` | 候補に表示された接続OKアンテナをすべて順番に使う |
| `q` | アンテナ設定を変更せずに進む |

### 順次切替時の表示例

```text
--- Inventory対象: ANT0（内蔵アンテナ） ---
コマンドモード用アンテナ設定はすでに ANT0 です。そのままInventoryに使用します。

--- Inventory対象: ANT1（外付けアンテナ1） ---
コマンドモード用アンテナ設定を一時変更します。
変更前: Ant0
変更後: ANT1（外付けアンテナ1）
FLASHは変更しません。
コマンドモード用アンテナ設定を一時変更しました。
```

ANT1の後にANT0へ戻る場合は、現在設定を追跡して明示的に戻します。

```text
--- Inventory対象: ANT0（内蔵アンテナ） ---
コマンドモード用アンテナ設定を一時変更します。
変更前: Ant1
変更後: ANT0（内蔵アンテナ）
FLASHは変更しません。
コマンドモード用アンテナ設定を一時変更しました。
```

### 終了時の復元

`q`でInventoryを終了すると、元のコマンドモード用アンテナ設定へ戻します。

```text
コマンドモード用アンテナ設定を元に戻します。
復元対象: Ant0
FLASHは変更していません。
コマンドモード用アンテナ設定を元に戻しました。
```

### 注意

- この処理は、現時点ではUTR-SUN02-4CHを対象にしています。
- 8CH機はアンテナ番号体系が異なるため、同じ処理を流用しないでください。
- FLASH書き込みは行いません。
- コマンドモード用パラメータだけを一時変更します。
- Ctrl+Cなどで強制終了した場合、復元処理まで到達しない可能性があります。確認作業では、原則として`q`で終了してください。

## よくあるトラブルと切り分け観点

### COM ポートが見えない

- USB ケーブルを確認する。
- Windows のデバイスマネージャーで COM ポートの認識状態を確認する。
- ドライバが必要な環境では、ドライバ導入状況を確認する。
- 別の USB ポートやケーブルで再確認する。
- UTRRWManager でも同じ COM ポートが見えるか確認する。
- Python サンプルを起動し直し、COM ポート一覧が更新されるか確認する。

### COM ポート選択で迷う

- 複数の COM ポートが表示された場合は、番号または `COM6` のような COM 名を入力する。
- `q` または `Q` を入力すると、実機接続前に終了できる。
- COM ポートが 1 件だけ見つかった場合は、そのポートを使うか確認される。
- 1 件だけ見つかった場合の確認では、Enter、`y`、`Y` で使用し、`n`、`N` で通常の選択へ進む。

### ACK が返らない

- COM ポート番号が正しいか確認する。
- 通信速度が機器設定と一致しているか確認する。
- 通信できない場合は UTRRWManager で現在の通信速度を確認する。
- 機器側が 19200 bps の設定なら、ボーレート入力で `19200` を明示入力する。
- 機器電源と USB 接続を確認する。
- コマンドモードが必要な操作では、先に `COMMAND_MODE_SET` の応答を確認する。
- 送信しているコマンドが設定変更系でないか、手順上必要なものか確認する。

### NACK が返る

- NACK表示に出る確認ポイントを最初に見る。
- UTRRWManagerで同じ接続条件、同じタグ条件を確認する。
- アンテナ接続、タグ距離、タグ向き、送信出力を確認する。
- Access Password、対象メモリバンク、メモリロック状態を確認する。
- Raw hex を共有する場合は、実測PC/UIIや顧客情報が含まれないようにマスクする。

### タグが読めない

- アンテナ接続を確認する。
- 送信出力設定を確認する。
- 送信出力レスポンス `F0 00` は `24.0 dBm` です。異常値が出る場合はエンディアン解釈を確認する。
- タグの向き、距離、枚数を変えて確認する。
- UTRRWManager で同じ条件の読み取り結果を確認する。
- `UHF_GET_INVENTORY_PARAM` で Inventory 設定を確認する。

### 読み取り枚数が合わない

- タグ同士の距離や重なりを確認する。
- アンテナからの距離を固定して再確認する。
- UTRRWManager と Python で同じ条件、同じ設定になっているか確認する。
- Inventory パラメータの差がないか確認する。

### RSSI 表示が UTRRWManager と違う

- まず `rssi_validation_plan.md` に沿って記録する。
- 1 回の測定だけで補正式を決めない。
- 距離、タグ種別、アンテナ、送信出力、Inventory 設定をそろえて比較する。
- 差分が一定か、条件で変動するかを確認する。

### アンテナ未接続エラー

- アンテナケーブルの接続状態を確認する。
- コネクタの緩みや接触不良を確認する。
- アンテナを交換できる場合は、同じ条件で再確認する。
- UTRRWManager 側でも同じエラーが出るか確認する。
