# RSSI Validation Plan

## 目的

RSSI 検証の目的は、Python 表示値と UTRRWManager 表示値の差を確認することです。

この段階では、補正式を先に決めません。複数条件の比較結果と仕様上の根拠を確認してから、コード変更が必要かを判断します。

## 現在の実装方針

- RSSI を符号付き 16bit として扱う。
- 値を 10 で割って表示用の値にする。
- 根拠なく補正しない。

## 検証時に記録する項目

| 項目 | 記録内容 |
|---|---|
| 日付 | 測定日 |
| 機種 | UTR-S201、UTR-SUN02-4CH、UTR-SUN02-8CH など |
| 接続方式 | USB シリアルなど |
| COM ポート | `COMx` のように記録 |
| 通信速度 | 例: `115200` |
| アンテナ | 使用アンテナをマスク可能な範囲で記録 |
| タグ種別 | タグ種別や形状 |
| 距離 | アンテナとタグの距離 |
| Python 表示 RSSI | Python 側の表示値 |
| UTRRWManager 表示 RSSI | UTRRWManager 側の表示値 |
| 差分 | Python 表示値と UTRRWManager 表示値の差 |
| 備考 | 条件差、再測定メモなど |

## 判断基準

次の観点で差分を確認します。

- 差が一定オフセットか。
- 距離やタグで変動するか。
- 符号、エンディアン、単位の問題か。
- 表示丸めの問題か。
