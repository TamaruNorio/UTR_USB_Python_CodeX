# 送信出力一時変更CLI 実機確認手順

この手順は、`src/utr_output_power_temporary_change_cli.py` の実機確認用です。

## 目的

UTR-SUN02-4CH / USM02 に対して、送信出力を一時変更し、確認後に元の送信出力へ復元できることを確認します。

## 重要な制約

- 対象は UTR-SUN02-4CH / USM02 のみです。
- FLASHは変更しません。
- Inventoryは実行しません。
- コマンドモード用パラメータだけを一時変更します。
- 終了時に、読み取り済みの元の送信出力へ復元します。
- 実IP、実タグID、PC+UII、顧客名、設置先名はログに残さないでください。

## 実行前確認

PowerShellで以下を実行します。

```powershell
cd "D:\My documents\Python Scripts\CodeX\UTR_USB_Python_CodeX"

git status --short
```

何も表示されないことを確認します。

## 実行コマンド

```powershell
py .\src\utr_output_power_temporary_change_cli.py
```

## 入力例

```text
送信出力[dBm]: 20.0
この内容で一時変更しますか？ [y/N]: y
確認が終わったら Enter を押してください。元の送信出力へ復元します。
```

## 確認ポイント

1. ROMバージョンが `2052USM02` として読めること。
2. 仕様書照合機種が `UTR-SUN02-4CH` になること。
3. 現在の送信出力設定が表示されること。
4. 変更用フレームと復元用フレームが表示されること。
5. `FLASHは変更しません。` と表示されること。
6. 変更後に送信出力値が変更後の値として再読み取りされること。
7. Enter後に元の送信出力へ復元されること。
8. 復元後に元の送信出力値として再読み取りされること。
9. 最後に `接続を閉じました。` と表示されること。

## 期待ログ例

実値を貼る場合は、以下のように機密情報を含まない範囲にしてください。

```text
ROMシリーズ名: USM02
仕様書照合機種: UTR-SUN02-4CH（内蔵1CH + 外付け3CH）
現在の送信出力: 24.0 dBm
変更後の送信出力: 20.0 dBm
FLASHは変更しません。
送信出力の一時変更がACKで完了しました。
送信出力値: 20.0 dBm
送信出力の復元がACKで完了しました。
送信出力値: 24.0 dBm
接続を閉じました。
```

## 異常時の扱い

復元に失敗した場合は、UTRRWManagerまたは再実行時の読み取りで、機器側の現在設定を確認してください。
