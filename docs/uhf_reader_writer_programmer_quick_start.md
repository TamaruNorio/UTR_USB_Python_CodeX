# UHFリーダライタ開発者向け Quick Start

## このドキュメントの対象者

このドキュメントは、次の人が UTR-S201/202 系リーダライタを USB 接続で確認するときの入口です。

- Python 初心者
- UTR-S201/202 を USB 接続で確認したい人
- UTR-SUN02-4CH の送信出力一時変更、アンテナ接続確認、ANT別読み取り確認をしたい人

## 事前準備

作業前に、以下を準備してください。

- Windows PC
- UTR-S201/202 系リーダライタ、または UTR-SUN02-4CH
- USB 接続
- アンテナ
- RF タグ
- UTRRWManager での事前確認

UTRRWManager で接続、ROM バージョン確認、タグ読み取りなどの基本動作を先に確認してから、Python サンプルでの確認に進んでください。

## 最初に確認すること

- COM ポートが Windows 上で認識されていること
- Python サンプルの COM ポート選択では、表示された番号または `COM6` のような COM 名を入力できること
- 通信速度入力で未入力なら 115200 bps になること
- 通信速度が機器設定と一致していること
- 機器電源が入っていること
- アンテナが正しく接続されていること
- 送信出力設定が想定どおりであること
- 送信出力値は `dBm * 10` を LSB/MSB の順で解析すること
- 読み取りに使うタグが準備されていること

## 実行コマンド

通常の確認は、PowerShellで以下を実行します。

```powershell
cd "D:\My documents\Python Scripts\CodeX\UTR_USB_Python_CodeX"
py .\src\utr_usb_sample.py
```

標準入口 `src/utr_usb_sample.py` は、送信出力一時変更つきInventoryフローを起動します。

## 標準フロー

標準フローは以下の順で進みます。

1. COMポートを選択する。
2. ボーレートを入力する。未入力なら 115200 bps。
3. ROMバージョンを読み取り、機種を確認する。
4. コマンドモードへ切り替える。
5. 現在の送信出力設定を読み取る。
6. UTR-SUN02-4CH / USM02 の場合、送信出力をInventory中だけ一時変更するか確認する。
7. Inventory前のリーダライタ設定とInventoryパラメータを表示する。
8. 必要に応じてアンテナ接続確認を行う。
9. Inventoryを実行する。
10. `q` でInventoryを終了する。
11. 送信出力を一時変更した場合は、元の値へ復元する。
12. アンテナ設定を変更した場合は、元のコマンドモード用アンテナ設定へ復元する。
13. 結果ファイルを保存し、接続を閉じる。

## 送信出力一時変更

UTR-SUN02-4CH / USM02 では、Inventory前に送信出力を一時変更できます。

入力例:

```text
送信出力をInventory中だけ一時変更しますか？ [y/N]: y
変更後の送信出力値を入力してください。例: 12.0 / 20.0 / 24.0
送信出力[dBm]: 20
この内容で一時変更してInventoryへ進みますか？ [y/N]: y
```

この処理では、以下を行います。

- 現在の送信出力値を読み取る。
- 現在のキャリア送信時間、キャリア休止時間、キャリアセンス待ち時間を読み取る。
- 変更用フレームを生成する。
- 復元用フレームを生成する。
- ユーザー確認後に、コマンドモード用パラメータだけを一時変更する。
- 変更後に送信出力を読み戻す。
- Inventory終了時に元の送信出力へ復元する。
- 復元後に送信出力を読み戻す。

重要:

- FLASHは変更しません。
- `UHF_SET_INVENTORY_PARAM` は自動送信しません。
- 送信出力変更はコマンドモード用パラメータだけを対象にします。
- 強制終了、USB切断、電源断では復元処理まで到達できない場合があります。確認作業では、原則として `q` で終了してください。

## UTR-SUN02-4CHのアンテナ確認と順次Inventory

UTR-SUN02-4CHでは、Pythonサンプル実行中にアンテナ設定表示・接続チェックを実行できます。

この機能は、以下を目的にしています。

- ANT0〜ANT3の接続状態を確認する。
- Inventoryに使えるアンテナだけを候補表示する。
- コマンドモード用アンテナ設定だけを一時変更する。
- FLASHを書き換えずに、終了時に元のコマンドモード設定へ戻す。
- 複数アンテナを同時指定ではなく、順次切替でInventoryする。

### 実行時の入力例

```text
アンテナ設定表示・接続チェックを実行しますか？ [y/N]: y
```

ANT0とANT1が接続OKの場合、次のように候補が表示されます。

```text
Inventoryに使用できるアンテナ候補:
[0] ANT0（内蔵アンテナ）
[1] ANT1（外付けアンテナ1）
複数選択できます。例: 0 / 1 / 0,1 / all
複数選択時は、同時使用ではなく順番にアンテナを切り替えてInventoryします。
```

入力例は以下です。

| 入力 | 動作 |
|---|---|
| `0` | ANT0だけでInventoryする |
| `1` | ANT1だけでInventoryする |
| `0,1` | ANT0でInventory後、ANT1へ切り替えてInventoryする |
| `all` | 候補に表示された接続OKアンテナをすべて順番に使う |
| `q` | アンテナ設定を変更せず、現在のコマンドモード用アンテナ設定でInventoryを続行する |

### 終了時の復元

`q`でInventoryを終了すると、元のコマンドモード用アンテナ設定へ戻します。

```text
コマンドモード用アンテナ設定を元に戻します。
復元対象: Ant0
FLASHは変更していません。
コマンドモード用アンテナ設定を元に戻しました。
```

## 実行前の注意

- 顧客情報、実シリアル番号、実タグ ID、未マスクの実測ログは記録や共有時に混入させないでください。
- Inventory結果は `inventory_results.txt`、`inventory_results.csv`、`inventory_results.json` に保存されます。
- Inventoryを1回以上実行した場合は、タグ0枚でも結果を保存します。Inventory自体を実行していない場合は保存しません。
- CSVはExcelでの確認向け、JSONは後続処理や比較用です。
- 複数アンテナ順次Inventoryでは、CSV/JSONにアンテナ番号、ラベル、説明も保存されます。
- 既存CSVのヘッダーが旧形式の場合は、旧CSVを `inventory_results_legacy_YYYYMMDD_HHMMSS.csv` に退避します。
- `inventory_results*` は実タグIDを含む可能性があるため、Git管理に入れません。
- 実測PC/UIIやログを公開する場合は、必ずマスクしてください。
- 実行時に `PC+UIIを画面表示でマスクしますか？` で `y` を選ぶと、画面表示だけ `PC+UII: 省略` にできます。集計キーと結果ファイルには実値を保持します。
- 読み取り前にブザー通知のON/OFFを選べます。ONの場合、タグありはピッピッピ、タグなしはピーで通知します。
- `q` で読み取りを終了しても、終了ブザーは鳴りません。

## よく使う確認

| 確認 | 用途 |
|---|---|
| `ROM_VERSION_CHECK` | リーダライタと通信できるか、機器応答が返るかを確認する |
| `COMMAND_MODE_SET` | コマンドモードへ移行できるかを確認する |
| `UHF_READ_OUTPUT_POWER` | 送信出力値とキャリア関連時間を確認する |
| `UHF_READ_FREQ_CH` | 周波数設定を確認する |
| `UHF_GET_INVENTORY_PARAM` | Inventory 関連パラメータを読み出して確認する |
| `UHF_INVENTORY` | タグ読み取り応答を確認する |
| `UHF_BUZZER_pipipi` | ACK 応答と実音で簡易動作を確認する |
| `UHF_CheckAntenna` | UTR-SUN02-4CH のANT0〜ANT3接続状態を確認する |
| アンテナ切替設定読み取り | コマンドモード用パラメータとFLASHデータのアンテナ設定を確認する |
| アンテナ切替設定書き込み | コマンドモード用パラメータだけを一時変更する。FLASHは変更しない |
| 出力設定書き込み | コマンドモード用パラメータだけを一時変更する。FLASHは変更しない |

## よくあるトラブルと切り分け観点

### COM ポートが見えない

- USB ケーブルを確認する。
- Windows のデバイスマネージャーで COM ポートの認識状態を確認する。
- ドライバが必要な環境では、ドライバ導入状況を確認する。
- 別の USB ポートやケーブルで再確認する。
- UTRRWManager でも同じ COM ポートが見えるか確認する。
- Python サンプルを起動し直し、COM ポート一覧が更新されるか確認する。

### ACK が返らない

- COM ポート番号が正しいか確認する。
- ボーレートが機器設定と一致しているか確認する。
- 別ソフトが同じ COM ポートを開いていないか確認する。
- UTRRWManagerで同じ条件で通信できるか確認する。
- 電源、USBケーブル、USB変換ドライバを確認する。

### 送信出力が復元されない

- プログラムを `q` で終了したか確認する。
- 強制終了、USB切断、電源断がなかったか確認する。
- UTRRWManagerで現在の送信出力設定を確認する。
- Pythonサンプルを再実行して、Inventory前の送信出力設定表示を確認する。

### Inventory結果ファイルをGitに入れそうになった

以下を確認します。

```powershell
git status --short
```

`inventory_results*` が表示された場合は、GitHubへ上げないでください。必要なら別フォルダへ退避してください。
