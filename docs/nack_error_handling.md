# UHF Gen2 NACK Error Handling

このメモは、UTR-S201 の UHF / ISO18000-63 / EPCglobal Class1 Generation2 通信で NACK 応答を受信したときの確認手順です。

## NACKとは

NACK は、リーダライタがコマンドを正常完了できなかったことを示す応答です。

ACK はコマンドが受け付けられたことを示します。NACK はエラーコードを含み、アンテナ接続、タグ応答、コマンド形式、メモリ操作条件などの切り分けに使います。

## NACKレスポンス形式

UTR-S201 通信プロトコル説明書 7.6 の NACK レスポンスは、次の形式です。

| 項目 | 内容 |
|---|---|
| STX | `02h` |
| アドレス | `00h` など |
| コマンド | `31h` |
| データ長 | `0Ah` |
| 詳細コマンド | エラーが発生した詳細コマンド |
| エラーコード1 | 主なエラー種別 |
| エラーコード2 | `CMD_UHF_IC_ERROR` 時の詳細 |
| エラーコード3 | UHF_Encode 実行時の詳細 |
| エラーコード4 | 追加情報 |
| 予約 | 5バイト |
| ETX | `03h` |
| SUM | SUM値 |
| CR | `0Dh` |

## エラーコード1

| コード | シンボル | 内容 | 主な確認ポイント |
|---|---|---|---|
| `01h` | `CMD_CRC_ERROR` | RFタグから受信したデータのCRC不一致 | タグ距離、タグ向き、周辺金属、アンテナ位置、再読み取り |
| `02h` | `CMD_TIME_OVER` | RFタグからの受信データが途中で途切れた | タグ距離、タグ枚数、電波環境、再実行 |
| `03h` | `CMD_RX_ERROR` | アンチコリジョン処理中にエラー | 複数タグ有無、タグ密集、Q値、再読み取り |
| `04h` | `CMD_RXBUSY_ERROR` | RFタグからの応答がない | タグ有無、タグ向き、距離、アンテナ接続、送信出力 |
| `07h` | `CMD_ERROR` | コマンド実行中にリーダライタ内部でエラー | コマンド条件、再実行、電源再投入 |
| `0Ah` | `CMD_UHF_IC_ERROR` | RFタグアクセス時、リーダライタ内蔵チップが返すエラー | エラーコード2、対象タグ、対象メモリバンク、Access Password |
| `60h` | `CMD_LBT_ERROR` | キャリアセンス時、タイムアウトでキャリアを送信できなかった | 電波環境、他機器、周波数チャンネル、再試行 |
| `64h` | `HARDWARE_ERROR` | ハードウェア内部異常 | 電源、再起動、機器交換、保守対応 |
| `68h` | `CMD_ANT_ERROR` | アンテナ断線検知時にRFタグ通信コマンドを送信した | アンテナ接続、ケーブル、接続CH、アンテナ設定 |
| `42h` | `SUM_ERROR` | 上位機器から送信されたコマンドのSUM値が不正 | コマンド生成、SUM計算、送信データ欠落 |
| `44h` | `FORMAT_ERROR` | 上位機器から送信されたコマンドのフォーマットまたはパラメータが不正 | データ長、詳細コマンド、パラメータ範囲、対象機種対応 |

## エラーコード2

エラーコード2は、主にエラーコード1が `0Ah`、つまり `CMD_UHF_IC_ERROR` の場合に確認します。

| コード | 内容 | 主な確認ポイント |
|---|---|---|
| `01h` | サポートされていません | コマンド種別、対象タグ、対象機種対応 |
| `02h` | 権限が不十分 | Access Password、対象メモリバンク、タグのロック状態 |
| `03h` | メモリオーバーラン | 対象メモリバンク、書き込み対象アドレス、データ長 |
| `04h` | メモリロック | 対象バンクがロック済みでないか、書き込み対象アドレス、Access Password |
| `05h` | 暗号違反 | Access Password、対象タグ設定、再実行 |
| `06h` | コマンドはカプセル化されません | コマンド種別、パラメータ、対象タグ |
| `07h` | レスポンスバッファオーバーフロー | 応答データ長、タグ枚数、再実行 |
| `08h` | セキュリティアウト | Access Password、タグ状態、再実行 |
| `0Bh` | 不十分な電力 | タグ距離、タグ向き、送信出力、アンテナ位置 |
| `0Fh` | 非特定のエラー | 再実行、タグ状態、通信条件 |
| `20h` | Writeに失敗 | 対象メモリバンク、書き込み対象アドレス、Access Password |
| `22h` | Killに失敗 | Kill Password、対象タグ、タグ状態 |
| `23h` | Lockに失敗 | Access Password、対象メモリバンク、ロック設定 |
| `80h` | 検出されない | タグ距離、タグ向き、アンテナ接続、送信出力、タグ種別 |
| `81h` | ハンドル取得失敗 | タグ距離、タグ応答、再実行 |
| `82h` | Accessパスワードエラー | Access Password設定、対象メモリバンク、タグのロック状態 |
| `90h` | CRCエラー | タグ距離、タグ向き、電波環境、再実行 |

上記以外のエラーコード2は、処理失敗として扱い、対象タグ、対象メモリバンク、Access Password、実行条件を確認します。

## UHF_Encode時のエラーコード3

| コード | 内容 | 主な確認ポイント |
|---|---|---|
| `01h` | Reserved領域への書き込み時にエラー | 対象領域、Access Password、ロック状態 |
| `02h` | EPC(UII)領域への書き込み時にエラー | EPC(UII)領域、書き込みデータ、Access Password、ロック状態 |
| `03h` | User領域への書き込み時にエラー | User領域、書き込み対象アドレス、Access Password、ロック状態 |
| `05h` | Lockコマンド発行時にエラー | Lock設定、Access Password、対象メモリバンク |

## 現場での切り分け手順

1. NACK表示の `Raw` を保存し、未マスクのまま公開しない。
2. `詳細コマンド` を確認し、どの操作で失敗したか確認する。
3. `エラーコード1` の分類と確認ポイントを見る。
4. `CMD_UHF_IC_ERROR` の場合は、エラーコード2を優先して確認する。
5. UHF_Encode 実行時は、エラーコード3も確認する。
6. UTRRWManagerで同じ条件を再現し、表示とログを比較する。

## よくある原因

- アンテナ未接続、接触不良、接続CH違い。
- タグ未検出、タグ距離、タグ向き。
- 周辺金属や電波環境による応答不安定。
- Access Password不一致。
- メモリバンクまたはアドレス指定ミス。
- ロック済みメモリへの操作。
- SUM不一致。
- FORMAT不正。
- LBTエラー。

## UTRRWManagerログとの比較方法

- Python側の `Raw` と UTRRWManager側ログのNACK応答を比較する。
- 詳細コマンド、エラーコード1、エラーコード2、エラーコード3が一致するか確認する。
- 接続CH、送信出力、Inventory条件、対象タグ条件をそろえる。
- 差分がある場合は、Python側の送信コマンドhexと UTRRWManager側ログを並べて確認する。

## 実測ログ共有時の注意

- 実測PC/UII、シリアル番号、顧客情報、実IP、未マスクログは共有用ファイルに含めない。
- タグ値は `E28011xxxxxxxxxxxxxx` のようにマスクする。
- Raw hex に実測タグ値や顧客固有条件が含まれる場合は、公開前にマスクする。
