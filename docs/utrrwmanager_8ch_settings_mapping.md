# UTRRWManager 8CH設定とPython制御の対応メモ

## 目的

このメモは、UTR-SUN02-8CH / USM08 を使うときの UTRRWManager 画面設定と、Python側の8CH順次Inventory CLIの制御範囲を整理するための資料です。

8CH機では、画面上の `ANT` 表示、使用アンテナ番号、内部アンテナ番号、外部アンテナ番号、アンテナ自動切換、アンテナ選択コマンドの関係が単純ではありません。

そのため、Python側では次の方針を固定します。

- UTRRWManager側の設定項目を勝手に変更しない。
- FLASHへ書き込まない。
- 送信出力を変更しない。
- 周波数を変更しない。
- `UHF_SET_INVENTORY_PARAM` を送信しない。
- 順次Inventory開始前に、変更前の使用アンテナ番号を読み取る。
- 終了時に、変更前の使用アンテナ番号へ自動復元する。

## 対象

```text
対象機種: UTR-SUN02-8CH / USM08
確認ソフト: UTRRWManager Ver.1.5.1.0
Python CLI: src/utr_8ch_sequential_inventory_cli.py
```

## UTRRWManager画面で確認した設定例

田丸さんの実機確認画面では、以下のような設定が確認されています。

```text
アンテナ自動切換: 有効
外部アンテナID出力: 無効
外部アンテナ接続数:
  アンテナ出力1: 1
  アンテナ出力2: 1
  アンテナ出力3: 1
  アンテナ出力4: 1
  アンテナ出力5: 1
  アンテナ出力6: 1
  アンテナ出力7: 1
  アンテナ出力8: 1
アンテナ選択コマンドを使用する: 無効
FLASHへの書き込み: チェックあり
```

注意点です。

- UTRRWManagerの画面で `FLASHへの書き込み` にチェックが入っていても、Python CLI側はFLASHへ書き込みません。
- Python CLI側は、コマンドモード用パラメータだけを一時的に変更します。
- UTRRWManager側の設定が結果表示、ANT列、読み取り枚数に影響する可能性があります。

## 用語整理

| 用語 | 意味 |
|---|---|
| 物理アンテナ | 実際の外付けアンテナ。UTR-SUN02-8CHでは ANT1〜ANT8。 |
| 内部アンテナ番号 | リーダライタ内部で扱うアンテナ番号。ANT1なら `00h`、ANT3なら `02h`。 |
| 外部アンテナ番号 | 外部アンテナIDや接続構成に関係する番号。通常0だけとは限らない。 |
| 使用アンテナ番号 | 内部アンテナ番号と外部アンテナ番号の組み合わせで表される番号。 |
| UHF_CheckAntenna番号 | 接続確認コマンドで指定する番号。ANT1なら `00h`、ANT3なら `02h`。 |
| UTRRWManagerのANT列 | 読み取り結果一覧に表示されるANT値。設定条件により単純な物理ANT番号とは限らない。 |

## Python CLIで使う主なコマンド

| 処理 | 詳細コマンド | Python側の目的 | FLASH影響 |
|---|---:|---|---|
| 使用アンテナ番号の読み取り | `48h` | 順次Inventory開始前に変更前設定を覚える | なし |
| 使用アンテナ番号の書き込み | `38h` | Inventory対象ANTへ一時切替、終了時に元へ自動復元 | なし |
| アンテナ接続確認 | `44h` | ANT1〜ANT8の接続OK/NG確認 | なし |
| Inventory | `UHF_INVENTORY` | タグ読み取り | なし |
| Inventoryパラメータ読み取り | `UHF_GET_INVENTORY_PARAM` | 現在設定の表示 | なし |

Python CLIでは、`UHF_SET_INVENTORY_PARAM` は送信しません。

## 重要な実機確認結果

PR #66 の実機確認では、順次Inventory開始前の変更前設定として以下が読み取られました。

```text
対象アンテナ: ANT3/EXT5
使用アンテナ番号: 44h
内部アンテナ番号: 02h
外部アンテナ番号: 04h
```

終了時には、次のフレームで自動復元できました。

```text
送信フレーム: 02 00 55 04 38 00 02 04 03 9C 0D
自動復元完了: ANT3/EXT5 / 使用アンテナ番号 44h
```

この結果から分かることは以下です。

- 変更前設定は、必ずしも `ANT1=00h` や `ANT3=40h` のような外部アンテナ番号0固定ではない。
- UTRRWManager側設定や外部アンテナ構成により、外部アンテナ番号が0以外になる可能性がある。
- そのため、終了時の復元は「ユーザーがANT番号を選ぶ方式」では不十分。
- 正しい復元方法は、開始前に読み取った内部アンテナ番号・外部アンテナ番号をそのまま書き戻す方式。

## UTRRWManagerのANT列について

UTRRWManagerの受信データ一覧には `ANT` 列があります。

確認画面では、以下のような値が表示されました。

```text
ANT列: 0
ANT列: 67
ANT列: 68
```

この `ANT` 列は、単純な物理アンテナ番号ではない可能性があります。

特に、以下の設定が関係する可能性があります。

- アンテナ自動切換
- 外部アンテナID出力
- 外部アンテナ接続数
- アンテナ選択コマンドを使用する/しない
- 内部アンテナ番号
- 外部アンテナ番号

したがって、現時点では次のように扱います。

```text
UTRRWManagerのANT列 = Python CLIの物理ANT番号と単純一致するとは限らない表示値
```

Python CLIの結果では、`確認対象アンテナ: ANT1 / 使用アンテナ番号 00h` のように、Pythonが意図して切り替えた対象を明示します。

## UTRRWManager設定とPython側の扱い

| UTRRWManager設定 | 画面上の状態例 | Python側で読むか | Python側で変更するか | 注意 |
|---|---|---:|---:|---|
| アンテナ自動切換 | 有効 | 現状は直接読まない | 変更しない | 読み取り結果やANT列に影響する可能性あり |
| 外部アンテナID出力 | 無効 | 現状は直接読まない | 変更しない | ANT列の見え方に関係する可能性あり |
| 外部アンテナ接続数 | 各出力1 | 現状は直接読まない | 変更しない | 外部アンテナ番号の扱いに関係する可能性あり |
| アンテナ選択コマンド | 無効 | 現状は直接読まない | 変更しない | 有効時はInventory対象指定の意味が変わる可能性あり |
| FLASHへの書き込み | チェックあり | 読まない | Python側では書かない | UTRRWManager側のチェックであり、Python側処理とは別 |
| 使用アンテナ番号 | ANT3/EXT5など | 読む | コマンドモード用だけ一時変更・復元 | Python側の安全ガード対象 |

## Python CLIの安全設計

現在のPython CLIは、以下の順で動作します。

```text
1. USB接続
2. ROMバージョン確認
3. コマンドモード切替
4. 送信出力・周波数・Inventoryパラメータを読み取り表示
5. 変更前の使用アンテナ番号を読み取り保持
6. ANT1〜ANT8の接続確認
7. 接続OKアンテナだけを候補表示
8. ユーザーがInventory対象ANTを選択
9. 選択順に使用アンテナ番号を書き込み
10. 各ANTでInventory
11. 終了時に、変更前の使用アンテナ番号へ自動復元
12. 必要に応じてANT別サマリを保存
13. シリアル接続を閉じる
```

変更前の使用アンテナ番号を読めない場合は、順次Inventoryを開始しません。

```text
変更前の使用アンテナ番号を保存できないため、順次Inventoryは開始しません。
理由: 終了時に元の設定へ自動復元できないためです。
```

## 現場確認時のログ取得ルール

公開・共有するログでは、次を守ります。

- PC+UIIはマスクする。
- シリアル番号は必要に応じてマスクする。
- COMポート番号は必要に応じて `COMx` へ置換する。
- 顧客名、設置場所、IPアドレスは載せない。
- UTRRWManagerのスクリーンショットを使う場合、タグIDや顧客情報が写っていないか確認する。

共有してよい形式の例です。

```text
PC+UII: 省略
RSSI: -31.7 dBm
確認対象アンテナ: ANT3 / 使用アンテナ番号 40h
```

## 今後の確認課題

以下は、まだ完全には整理しきれていないため、今後の実機確認項目です。

- UTRRWManagerのANT列 `0`、`67`、`68` の意味。
- アンテナ自動切換が有効/無効のときのPython CLI読み取り結果差分。
- 外部アンテナID出力が有効/無効のときのANT列の変化。
- アンテナ選択コマンドを使用する場合のInventory対象指定の挙動。
- 外部アンテナ接続数が1以外の場合の使用アンテナ番号読み取り結果。
- UTR-SUN02V-8CHでの内部アンテナ番号・外部アンテナ番号の違い。

## 現時点の結論

8CH機では、物理ANT番号だけを見て制御すると不十分です。

Python CLIでは、以下を分けて扱います。

```text
物理ANT番号: ANT1〜ANT8
内部アンテナ番号: 00h〜07h
外部アンテナ番号: 00h以上
使用アンテナ番号: 内部アンテナ番号と外部アンテナ番号の組み合わせ
UTRRWManager ANT列: 設定依存の表示値
```

終了時復元は、ユーザーがANT番号を指定するのではなく、開始前に読み取った変更前設定へ自動復元する方式を正とします。
