# UTR-SUN02-8CH 複数アンテナ順次Inventory確認手順

## 目的

この手順は、UTR-SUN02-8CH / USM08 で、接続OKアンテナを複数選択し、選択順に使用アンテナ番号を切り替えながらInventoryするための確認手順です。

主な確認ポイントは以下です。

- ANT1〜ANT8の接続状態を確認する。
- 接続OKアンテナだけをInventory候補にする。
- `1`、`1,3`、`all`、`q` の入力で対象アンテナを選ぶ。
- 選択順に使用アンテナ番号設定フレームを送信する。
- ACKならInventoryする。
- NACKまたはACK/NACKなしなら、そのANTはスキップする。
- ANT別に読み取り枚数、Inventory実行回数、スキップ回数を表示する。
- 終了時に、必要に応じて指定ANTへ使用アンテナ番号を戻す。

## 対象機種

```text
UTR-SUN02-8CH / USM08
```

ROM確認例です。

```text
ROMバージョン: 2120USM08
ファームウェアバージョン: 2.120
ROMシリーズ名: USM08
仕様書照合機種: UTR-SUN02-8CH（外付け8CH）
```

## 実行コマンド

```powershell
py .\src\utr_8ch_sequential_inventory_cli.py
```

## 実行前の確認

- UTR-SUN02-8CHの電源が入っていること。
- USBシリアル接続がPCから認識されていること。
- 対象COMポートが分かっていること。
- ボーレートが分かっていること。未入力時は 115200 bps を使用します。
- ANT1〜ANT8のうち、確認したいアンテナを接続していること。
- RFタグを読み取り可能な位置に置いていること。
- 公開ログではPC+UIIをマスクすること。

## 安全ガード

このCLIでは、以下を行いません。

- FLASH書き込み
- 送信出力変更
- 周波数変更
- `UHF_SET_INVENTORY_PARAM` 送信
- 接続エラーANTへの強制Inventory
- 8CH設定の永続保存

使用アンテナ番号設定は、コマンドモード用パラメータへ送信します。FLASHには保存しません。

## 8CHのアンテナ番号体系

UTR-SUN02-8CHでは、接続確認に使う番号と、使用アンテナ番号設定に使う番号の見え方が異なります。

### UHF_CheckAntennaの対象番号

| ANT | UHF_CheckAntenna番号 |
|---|---|
| ANT1 | `00h` |
| ANT2 | `01h` |
| ANT3 | `02h` |
| ANT4 | `03h` |
| ANT5 | `04h` |
| ANT6 | `05h` |
| ANT7 | `06h` |
| ANT8 | `07h` |

### 使用アンテナ番号

| ANT | 使用アンテナ番号 |
|---|---|
| ANT1 | `00h` |
| ANT2 | `20h` |
| ANT3 | `40h` |
| ANT4 | `60h` |
| ANT5 | `80h` |
| ANT6 | `A0h` |
| ANT7 | `C0h` |
| ANT8 | `E0h` |

実機確認済みのフレーム例です。

```text
ANT1: 02 00 55 04 38 00 00 00 03 96 0D
ANT3: 02 00 55 04 38 00 02 00 03 98 0D
```

## 基本操作手順

### 1. CLIを起動する

```powershell
py .\src\utr_8ch_sequential_inventory_cli.py
```

### 2. COMポートを選択する

表示例です。

```text
利用可能なCOMポート:
  [0] COM6
  [1] COM5
  [2] COM1
  [3] COM7
接続するCOMポートの番号またはCOM名を入力してください（終了は'q'）: 3
```

### 3. ボーレートを入力する

未入力なら 115200 bps です。

```text
ボーレートを入力してください（例: 19200, 115200, 未入力なら115200）:
接続成功: COM7 @ 115200bps
```

### 4. ROM確認とリーダ設定表示を確認する

```text
USB通信: OK（ROMバージョン ACK 受信）
ROMバージョン: 2120USM08
仕様書照合機種: UTR-SUN02-8CH（外付け8CH）
コマンドモードに切り替えました。
```

Inventory前に、送信出力、周波数、Inventoryパラメータを読み取り表示します。ここでは設定変更は行いません。

```text
UHF_SET_INVENTORY_PARAM は自動送信しません。設定変更は行いません。
```

### 5. アンテナ接続チェックを確認する

実機確認例です。

```text
アンテナ接続チェック結果:
  接続OK: ANT1, ANT3
補足: この時点ではアンテナ設定を書き換えていません。
```

### 6. Inventory対象ANTを選ぶ

接続OKアンテナだけが候補に表示されます。

```text
[1] ANT1（使用アンテナ番号: 00h）
[3] ANT3（使用アンテナ番号: 40h）
選択例: 1 / 1,3 / all / q
```

入力例です。

| 入力 | 動作 |
|---|---|
| `1` | ANT1だけでInventoryする |
| `3` | ANT3だけでInventoryする |
| `1,3` | ANT1、ANT3の順にInventoryする |
| `all` | 候補に表示されたANTを順にすべて使う |
| `q` | 対象選択を中止する |

### 7. dry-run表示を確認する

選択したANTごとに、使用アンテナ番号設定フレームが表示されます。

```text
[1/2]
対象アンテナ: ANT1
使用アンテナ番号: 00h
内部アンテナ番号: 00h
外部アンテナ番号: 00h
dry-run送信フレーム: 02 00 55 04 38 00 00 00 03 96 0D

[2/2]
対象アンテナ: ANT3
使用アンテナ番号: 40h
内部アンテナ番号: 02h
外部アンテナ番号: 00h
dry-run送信フレーム: 02 00 55 04 38 00 02 00 03 98 0D
```

内容に問題がなければ、順次Inventory開始確認で `y` を入力します。

```text
この内容で順次Inventoryを開始しますか？ [y/N]: y
```

### 8. Inventory条件を入力する

```text
読み取り結果をブザーで通知しますか？ [y/N]: y
PC+UIIを画面表示でマスクしますか？ [y/N]: y
選択ANT一巡を繰り返す回数を入力してください（1〜100、終了は'q'）: 2
```

公開ログに使う場合は、PC+UII画面表示マスクで `y` を選んでください。

### 9. 順次Inventory結果を確認する

実機確認例です。

```text
=== 8CH複数アンテナ順次Inventory結果まとめ ===
選択順: ANT1 -> ANT3
Inventory実行回数: 4
合計読み取り時間: 0.56 秒
合計読み取り枚数: 29 枚
ANT別結果:
  ANT1: 読み取り 2 枚 / Inventory実行 2 回 / スキップ 0 回
  ANT3: 読み取り 27 枚 / Inventory実行 2 回 / スキップ 0 回
```

## 終了時の使用アンテナ番号戻し

Inventoryループを `q` で終了すると、最後に使用したANTが表示されます。

```text
最後に使用したANT: ANT3
終了前に使用アンテナ番号を戻しますか？ [y/N]: y
```

`y` を選ぶと、今回選択した接続OKアンテナだけが戻し先候補として表示されます。

```text
[1] ANT1（使用アンテナ番号: 00h）
[3] ANT3（使用アンテナ番号: 40h）
選択例: 1 / 3 / q
戻し先は1つだけ入力してください。all と複数指定は使用できません。
```

ANT1へ戻す場合は `1` を入力します。

```text
終了前に戻すANT番号を入力してください（中止は'q'）: 1
```

送信前に、戻し用のフレームが表示されます。

```text
=== 終了前 使用アンテナ番号戻し dry-run ===
対象アンテナ: ANT1
使用アンテナ番号: 00h
内部アンテナ番号: 00h
外部アンテナ番号: 00h
dry-run送信フレーム: 02 00 55 04 38 00 00 00 03 96 0D
```

内容に問題がなければ `y` を入力します。

```text
この内容で使用アンテナ番号を戻しますか？ [y/N]: y
```

成功時の表示です。

```text
--- 終了前 使用アンテナ番号戻し: ANT1 ---
送信フレーム: 02 00 55 04 38 00 00 00 03 96 0D
戻し完了: ANT1 / 使用アンテナ番号 00h
接続を閉じました。
```

## NACKまたは応答なしの場合

使用アンテナ番号設定でNACKまたはACK/NACKなしの場合、そのANTはスキップします。

```text
使用アンテナ番号設定コマンド: NACK
ANTx はスキップします。
```

終了時戻しでNACKまたはACK/NACKなしの場合は、戻し失敗として表示して終了します。

```text
戻し失敗: NACK
```

または、

```text
戻し失敗: ACK/NACKなし
```

この場合は、必要に応じて再度CLIを実行し、戻したいANTを選択してください。

## 注意点

- 実機ログを公開する場合、PC+UIIは必ずマスクしてください。
- シリアル番号や顧客情報を公開ログに含めないでください。
- FLASHは変更しませんが、実行中のコマンドモード用パラメータは切り替わります。
- 終了時戻しを行わない場合、最後に使用したANT側の設定で終了する可能性があります。
- 強制終了した場合、終了時戻し処理まで到達しない可能性があります。
- UTR-SUN02V-8CHは内部アンテナ番号体系が異なるため、このCLIの実送信対象にはしません。
