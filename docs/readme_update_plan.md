# README更新案

## README更新の目的

README.md を、USB版サンプルの利用者が現在の状態を誤解しない形に更新するための改訂案です。

主な目的は以下です。

- このリポジトリの公開サンプルとしての位置づけを明確にする。
- 実機通信が未確認であることを明記する。
- `src/utr_usb_sample.py` は元サンプルとして温存していることを説明する。
- 追加した純粋関数モジュールとテストの位置づけを説明する。
- 開発チェック手順を短く案内する。

## 現在のREADMEに不足していそうな情報

- 元サンプルと追加モジュールの関係。
- 実機通信未確認の状態。
- `src/utr_protocol.py`、`src/utr_inventory.py`、`src/utr_commands.py` の役割。
- `tests/` が実機通信なしの確認用であること。
- `dev_check.ps1` と `git_preflight.ps1` の使い方。
- 実機ログ、顧客名、シリアル番号などを公開しない注意。
- 公開前のレビューと安全確認の流れ。

## 追加した方がよい章構成

1. プロジェクト概要
2. このリポジトリの位置づけ
3. 現在の状態
4. フォルダ構成
5. 主なファイル
6. セットアップ
7. テスト実行
8. 実機確認前の注意
9. セキュリティ・機密情報の注意
10. 今後の作業候補
11. ライセンスと無保証について

## 公開リポジトリとして見せる際の注意

- 開発経緯を書きすぎず、利用者に必要な情報を優先する。
- 利用者に必要な実行手順と注意事項を優先する。
- 実機未確認の内容は、確認済みのように書かない。
- 実機ログ、実IP、顧客名、シリアル番号などは含めない。
- 無保証サンプルであることを明確にする。

## 明記方針

| 項目 | READMEでの書き方 |
|---|---|
| 実機通信未確認 | 現時点ではUSB実機通信確認前であることを明記する |
| `src/utr_usb_sample.py` | 元サンプルとして未変更で温存していることを書く |
| `src/utr_protocol.py` | フレーム、SUM、NACKなどの純粋ヘルパーと説明する |
| `src/utr_inventory.py` | Inventory応答解析の純粋ヘルパーと説明する |
| `src/utr_commands.py` | 固定コマンド定義とフレーム生成ヘルパーと説明する |
| `tests/` | 実機通信なしで確認できるpytest群と説明する |
| `dev_check.ps1` | ローカル開発確認コマンドとして説明する |
| `git_preflight.ps1` | コミット/PR前の確認コマンドとして説明する |
| 公開前レビュー | 差分、実機確認状況、機密情報の有無を確認する |
